# UI/디자인 변경 룰

## 🎨 브랜드 컬러 원칙

### 절대 규칙
- **브랜드 메인 컬러: #5BC500 (초록색)**
- 모든 CTA 버튼은 초록색 사용
- 사용자가 명시적으로 요청하지 않는 한 **절대 색상을 임의로 변경하지 않음**

### 허용되는 색상
- 메인 버튼: `linear-gradient(135deg, #5BC500 0%, #4a9f00 100%)`
- 보조 버튼: 흰색 배경 + 회색 테두리 `#e5e7eb`
- 텍스트: 다크 그레이 `#374151`, 미디엄 그레이 `#6b7280`

## 📐 레이아웃 변경 원칙

### 1. 최소 변경 원칙 (Minimal Change Principle)
사용자가 "A를 수정해줘"라고 하면:
- ✅ **A만 수정**
- ❌ A, B, C를 함께 수정하지 않음
- ❌ 요청하지 않은 구조 변경 금지

**예시:**
- 요청: "버튼 레이아웃 예쁘게" → 버튼 스타일만 개선
- 하지 말아야 할 것: 색상 변경, 전체 레이아웃 재구성, 중앙 정렬 추가 등

### 2. 점진적 개선 (Incremental Improvement)
- 한 번에 많은 것을 바꾸지 않음
- 작은 변경 → 피드백 → 조정
- 과도한 교정 금지 (예: 작다고 해서 2배로 키우지 않음)

### 3. 일관성 유지 (Consistency First)
- 같은 타입의 컴포넌트는 같은 스타일 사용
- 예: 폼 제출 버튼 = 성공 화면 버튼
- 기존 디자인 시스템과 일관성 유지

## 🎯 구체적 실행 규칙

### 버튼 스타일링 시
1. 기존 버튼 스타일 먼저 확인
2. 브랜드 컬러(#5BC500) 사용
3. 일관된 패딩/폰트 크기 적용
4. 사용자가 색상 변경을 명시하지 않으면 초록색 유지

### 레이아웃 조정 시
1. "여유있게" = 마진/패딩만 조정
2. 구조 변경은 명시적 요청이 있을 때만
3. maxWidth, 중앙 정렬 등은 요청받았을 때만 추가

### 모달/팝업 디자인 시
1. 기존 모달과 일관성 유지
2. 패딩: `2rem ~ 2.5rem` (기본)
3. 버튼 간격: `0.75rem ~ 1rem`
4. 제목-내용-버튼 간격: 충분히 확보

## ⚠️ 금지 사항

### 절대 하지 말 것
1. **브랜드 컬러 임의 변경**
   - ❌ 보라색, 파란색 등으로 변경
   - ✅ 항상 #5BC500 사용

2. **과도한 디자인 변경**
   - ❌ 요청 범위를 넘어서는 변경
   - ❌ "예쁘게"를 이유로 전체 재디자인

3. **불필요한 구조 변경**
   - ❌ flexbox → grid 같은 구조 변경
   - ❌ 중앙 정렬, maxWidth 등 요청 없는 제약 추가

- [ ] 기존 컴포넌트와 일관성이 있는가?
- [ ] 과도하게 변경하지 않았는가?
- [ ] 구조를 불필요하게 바꾸지 않았는가?
